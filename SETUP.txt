ft-tracklink — Setup and Installation
=====================================

Prerequisites
-------------

- Node.js 18+ and npm
- Forge CLI (npm install -g @forge/cli)
- Atlassian Cloud account with developer access

Installation
------------

1. Install dependencies:

   npm install
   cd resources/track-viewer
   npm install
   cd ../..

2. Build the Frame component:

   cd resources/track-viewer
   npm run build
   cd ../..

3. Update manifest.yml:
   - Replace REPLACE-WITH-APP-ID in manifest.yml with your app ID
     (generated by forge create or from Atlassian Developer Console)

4. Set your Forge app ID in manifest.yml (full ARI), e.g.:

   ari:cloud:ecosystem::app/<your-uuid>

   From Developer Console or: forge register

5. Log in (once per machine):

   forge login

6. Build and deploy to development:

   npm run forge:deploy

   Or manually: npm run build && forge deploy -e development

7. Install on your Jira site (first time):

   npm run forge:install

   Or: forge install -e development

   Select Jira and your site (e.g. fat-tails.atlassian.net).

8. After UI changes (Phase 2+), redeploy and upgrade:

   npm run forge:deploy
   npm run forge:install:upgrade

Phase 2 (ADS UI Kit) — verify in Jira
-------------------------------------

- Global page: Apps → Formula 1 Track Linker (DEVELOPMENT)
- Issue action: open any issue → Track Linker
- Toggle Jira light/dark theme; chrome outside the map should follow ADS tokens
- Pan/Brush controls appear above the map (not inside the Frame)
- Upload modal, empty states, and selection/create panels use SectionMessage styling
